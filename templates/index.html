Exercise 1

We'd like to create a REST API for our Data Science students. Create a flask application that has three views that each take one of the following methods: 'GET', 'POST', 'DELETE'. Set up a similar REST API, storing your data to an dictionary. Your API should allow GET requests that return the dictionary, allow POST requests that add students and DELETE requests that remove students from the dictionary.

Add your code to it's own github repo and submit the link to the repo.

My solution

GET requests displayed all DS students can be made at: 'localhost:5000/view'

POST requests to add a new student can be made at: 'localhost:5000/add'

DELETE requests to remove a student by typing their name can be made at: 'localhost:5000/remove'
